# Tester update - v2.21.12

Please move to MyLocalAssistant 2.21.12.0 for this hotfix cycle.

## Why another hotfix

The previous release fixed one admin redirect path, but a second redirect path (session-expiry and some login-entry flows) could still drop users into chat after login.

## What changed in v2.21.12.0

- Preserved requested path through login using both query and state fallback.
- Preserved current destination during 401/session-expiry redirects.
- Result: opening `/admin` and logging in as owner/admin should consistently return to `/admin`.

## What to test

- Tray icon -> Open Admin from logged-out state:
  - log in with owner
  - verify landing page is `/admin`, not chat
- Session-expiry path:
  - open `/admin`
  - force an expired session (restart server or sign out/in flow)
  - log in again
  - verify return path is still `/admin`
- Non-admin behavior:
  - open `/admin` with non-admin user
  - verify explicit admin-required screen appears

## Update steps

- Installed build: tray icon -> Check for updates.
- Fresh install: use the 2.21.12.0 installer once published.
- After update, verify version in tray About window is 2.21.12.0.

## What to report back

- exact version shown in About
- exact navigation path taken (for example: tray Open Admin -> login -> final URL)
- expected vs actual result
- screenshots and timestamps for any mismatch
